01 — Foundation

What is OpenUI?

A full-stack Generative UI framework. LLMs output a declarative language instead of JSON or HTML. The runtime handles all interactivity without calling the LLM again.

LLM OutputOpenUI Lang
LexerToken stream
ParserStreaming AST
MaterializerResolve refs
Evaluator$vars, Query()
RendererReact Components

Key Insight

Generate once, execute forever. The LLM produces a declarative program with Query(), Mutation(), $variables, and Action(). The runtime handles data fetching, form interactions, and re-rendering with zero additional LLM calls.

defineComponent()

Register components with Zod v4 schema + React renderer

createLibrary()

Auto-generates system prompts teaching the LLM your components

<Renderer />

Standalone embed — no chat interface required

02 — Opportunities

Integration Opportunities

Seven ways OpenUI can transform the agent orchestration experience, ranked by impact.

01 Agent-Generated Insight Cards
High ImpactMedium
▸
Agents produce rich, interactive summaries of their work instead of plain-text strings. Decision logs, file change maps, test results, and cost metrics — all generated by the agent's LLM using OpenUI components.

Each card is unique because the LLM generates it from what actually happened in the session.

Modified 4 files, added 12 tests --- Files Modified --- + src/auth/refresh.ts +47 -12 + src/auth/middleware.ts +8 -3 + tests/auth.test.ts +89 [new] Cost: $0.47 Duration: 12m Commits: 3
02 Interactive Session Timeline
High ImpactMedium
▸
The dashboard shows state (working/done) but not the journey. Build an OpenUI timeline from lifecycle events: spawned, analyzed, coded, tests failed, fixed, PR created, CI passed, merged.

Timelines are inherently variable — some sessions have 5 events, others 50. The LLM generates contextually appropriate layouts each time.
10:32 ● Spawned — issue #1234 | 10:33 ● Analyzed — read 4 files | [+ Root cause reasoning] | 10:35 ● Implemented — refresh.ts | [+ View diff] | 10:38 ✗ Tests failed — 2 failures | 10:40 ● Fix applied | 10:42 ✓ PR created — #456 | 11:02 ✓ Merged
03 Agent Copilot Panel
High ImpactHard
▸
An OpenUI-powered panel alongside the terminal. Ask natural-language questions — "what files did you change?", "why did CI fail?" — and get interactive UI responses, not text. File trees are expandable, diffs are navigable, checklists have progress bars.

This is OpenUI's sweet spot: natural-language queries producing interactive UIs.

06 — Start Here

Recommended Starting Point

Phase 1: On-Demand Analytics. Lowest risk, highest signal, validates the entire integration path.

Why start with analytics?

It's a standalone modal — zero changes to the existing dashboard, SSE data flow, or terminal. It proves OpenUI works with AO's design system and validates the LLM-to-Renderer pipeline. Users immediately see the power of generated UIs. And it takes only 2-3 days.

Step 1
Add @openui/react to packages/web
Step 2
Create analytics component library with AO design tokens
Step 3
Build POST /api/analytics/generate route with LLM call
Step 4
Create AnalyticsModal.tsx with <Renderer />
Step 5
Add "Generate Report" button to Dashboard.tsx
Step 6
Ship it. Iterate based on what users ask for.
