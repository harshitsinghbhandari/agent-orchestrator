# Sukuna Synthesis: Building the King of AI Agents

To build the **Sukuna of AI Agents**—a system with "Domain Expansion" level control over codebases—we must synthesize the **raw power and speed of the current Orchestrator** with the **transcendental intelligence and structure of APEX**.

## 🔴 The Core Philosophy: "Cleave and Dismantle"
Like Ryomen Sukuna's innate techniques, the ultimate agent must be able to:
1. **Dismantle** (L3/L4): Instantly decompose a massive monolith into a DAG of independent, executable tasks.
2. **Cleave** (L5/L6): Execute these tasks with surgical precision in parallel, then "cleave" through bugs using an internal multi-tier critic pipeline.

---

## 🛠️ The Synthesis Strategy

### 1. Hybrid Orchestration (AO + APEX L3/L4)
Instead of starting from scratch, we inject **APEX L3/L4 (DAG-Native Planning)** into the **Current AO's tmux/worktree runtime**.
- **Result**: We get the speed of AO’s local execution but with the structural intelligence of a dependency-aware graph.
- **Why?**: Tmux and Worktrees are the "body" (reliable and fast); DAG-planning is the "mind".

### 2. The Persistent Brain (APEX L2) as the "Soul"
We must wrap the current AO sessions with an **L2 Context Layer** (the `embed.py` direction).
- **Implementation**: Before an agent starts in AO, we query the "Brain" for historical failures and architectural patterns.
- **Outcome**: The agent doesn't just run tools; it *learns* from previous sessions. Every "Dismantle" action gets smarter over time.

### 3. Progressive Sovereignty (L6/L7)
We don't need a full 9-layer system for every task. We implement **Adaptive Gating**.
- **Simple Tasks**: Use AO's current "lightweight" flow.
- **Complex/High-Risk Tasks**: Trigger the **L6 Multi-Tier Critic** and **L7 Human Gates**.
- **Outcome**: Minimizes resource overhead while maintaining high fidelity for critical changes.

### 4. Perception-Enabled Core (APEX L1)
We move from "Agent finds code" to "System provides code".
- **Implementation**: Build a permanent knowledge graph of the repository (L1) that is shared across all AO sessions.
- **Benefit**: No more "searching" for files. The agents operate with "Omniscient Context" from the start.

## 📈 Summary Roadmap: The Path to Sukuna
1. **Step 1 (The Body)**: Keep AO's Worktree/Dashboard/CLI as the stable foundation.

4. **Step 4 (The Eyes)**: Integrate L1 Perception (knowledge graph) to eliminate context-window limitations.
5. **Step 5 (The Judgment)**: Activate the L6 Critic Pipeline for high-risk nodes in the DAG.

**Result**: An agent system that is faster than the current AO, more intelligent than raw Claude Code, and more structured than any traditional orchestrator. **The King of Curses for the Codebase.**
